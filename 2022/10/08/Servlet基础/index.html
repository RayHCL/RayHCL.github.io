Servelet简介

​ Java Servlet是运行在Web服务器或应用服务器上的程序，它作为Web浏览器或其他HTTP客户端的请求和HTTP服务器上的数据库或应用程序之间的中间层。

1、Servlet执行的任务

  • 读取客户端（浏览器）发送的显式的数据。这包括网页上的 HTML 表单，或者也可以是来自 applet 或自定义的 HTTP 客户端程序的表单。
  • 读取客户端（浏览器）发送的隐式的 HTTP 请求数据。这包括 cookies、媒体类型和浏览器能理解的压缩格式等等。
  • 处理数据并生成结果。这个过程可能需要访问数据库，执行 RMI 或 CORBA 调用，调用 Web 服务，或者直接计算得出对应的响应。
  • 发送显式的数据（即文档）到客户端（浏览器）。该文档的格式可以是多种多样的，包括文本文件（HTML 或 XML）、二进制文件（GIF 图像）、Excel 等。
  • 发送隐式的 HTTP 响应到客户端（浏览器）。这包括告诉浏览器或其他客户端被返回的文档类型（例如 HTML），设置 cookies 和缓存参数，以及其他类似的任务。

2、Servlet生命周期

  • 将第一个达到服务器的HTTP请求委派到Servlet容器。

  • init()：进行初始化，在一整个servlet生命周期，只会被执行一次，条件不同，执行的时间不同。

  • service()：请求servlet调用service()方法来处理客户端的请求，并在适当的时候调用doGet、doPost、doPut、doDelete等方法。

  • destroy()：调用该方法终止（结束）。

  • 由JVM垃圾回收器对servlet进行垃圾回收。

5、请求头信息

​ Servlet需要处理基于Http的请求，接收到的请求是什么样的格式？

一个HTTP请求报文由请求行、请求头、空行和请求数据4个部分组成。

  • 请求行

    请求方法+空格+URL+空格+HTTP协议版本号

    请求方法有：GET、POST、HEAD、PUT、DELETE、OPTIONS、TRACE、CONNECT。

    • GET: 使用GET方法时，请求参数和对应的值附加在URL后面，利用一个问号？代表URL的结尾与请求参数的开始，传递参数长度受限制。目前说的GET长度有限制，是特定的浏览器及服务器对它的限制
    • POST:POST方法将请求参数封装在HTTP请求数据中，以k-v形式出现。理论上讲，post是没有大小限制的，Tomcat下默认post长度为2M，可通过修改conf/server.xml中的“maxPostSize=0”来取消对POST大小的限制。

  • 请求头

    请求头部由关键字/值对组成，每一行一对，关键字和值之间用冒号(:)分隔。

    协议头 说明 实例
    Accept 可接收的响应内容类型（Content-Types） Accept: text/plain
    Accept-Charset 可接收的字符集 Accept-Charset: utf-8
    Accept-Encoding 可接收的响应内容的编码方式 Accept-Encoding: gzip,deflate
    Connection 客户端想要优先使用的连接类型 Connection: keep-alive Connection: Upgrade
    Cookie 由之前服务器通过设置set-Cookie设置的一个HTTP协议Cookie Cookie: $Version=1;Skin=new;
    Date 发送该消息的日期和时间 Wed, 18 May 2022 14:51:39 GMT
    Export 表示客户端要求服务器做出特定的行为
    From 发送此请求的用户的邮件地址
    Host 表示服务器的域名以及服务器所监听的端口号，如果请求的端口号是80，可省略
    Upgrad 要求服务器升级到一个高版本协议
    Via 告诉服务器，这个请求是由那些代理发出的
    Warning 一个一般性警告，表示在实体类中可能存在错误

  • 空格

    最后一个请求头之后是空格，发送回车符和换行符，通知服务器以下不再有请求头。

  • 请求数据

    请求数据在POST方法中使用，与请求数据相关的最常用的请求头是Content-Type和Content-Length。

6、响应头信息

​ 响应也有一定格式，响应与请求的唯一区别在于第一行中用状态信息代替了请求信息。状态行通过提供一个状态码来说明所有请求的资源情况。

  • 状态行

状态行包括协议版本号、状态码、状态描述。其中状态码有3位数字组成，表示请求是否被理解或满足。

状态码 说明
200 响应成功
301 永久重定向，搜索引擎将删除源地址，保留重定向地址
302 暂时重定向，重定向地址由响应头中的Location属性指定，由于搜索引擎的判定问题，较为复杂的URL容易被其他网站使用更为精简的URL以及302重定向劫持。
400 客户端请求有语法错误，不能被服务器识别
404 请求资源部存在
500 服务器内部错误
  • 响应头

响应头是有k-v组成

响应头 说明 实例
Age 响应对象在代理缓存中存在的时间，以s为单位 Age: 12
Allow 对于特定资源的有效动作
Connection 针对连接所预期选项
Date 消息的日期和时间
Expires 指定一个日期、时间，超过该时间则认为此回应已经过期
Link 用来表示和另一个资源之间类型关系，此关系类型是在RFC5988定义
Location 用于进行重定向，或创建了某个新资源时使用
Server 服务器名称
Set-Cookie 设置HTTP cookie
Status 通过网关接口的响应头字段，用来说明当前Http连接响应状态
Upgrade 要求客户端升级到另一个高版本协议
Via 告知代理服务器的客户端，当前响应是通过什么途径发送的
Warning 一般性警告，告知在实体内可能存在错误
  • 空行

与请求一样，响应头和响应体必须由空行连接

  • 响应体

响应体主要包含服务器对于浏览器请求的响应消息，该响应可以是一个HTML文档。

7、Cookie

​ cookie是用来在浏览器端存储用户的状态信息，然后再访问后端时将这部分信息带回到后端。

工作原理：

（1）、发送请求时：浏览器检查所有存储的Cookie，如果某个Cookie所声明的作用范围大于等于将要请求的资源所在位置，则把该cookie附在请求资源的HTTP请求头上发送给服务器。

（2）、处理请求时：在服务器端，一般会对请求头中带的Cookie信息做检查（比如登录检查），如果检查通过，才能进行实际的业务处理。

（3）、如果校验不通过，比如没有找到Cookie或者Cookie信息不正确，跳转让其登录，登录完成之后，在响应头中返回cookie,浏览器会根据返回的Cookie信息，保存在硬盘或内存中以供下次使用。

8、Session

由于HTTP是无状态协议，所以服务器需要记录用户的状态时，就需要用某种机制来识别具体的用户，这个机制就是session。

​ session机制是一种服务器端的机制，使用一种类似散列表的结构来保存信息。例如购物车，点击下单时，由于HTTP无状态，所以不知道是哪个用户操作的，所以服务端要为特定的用户创建特定的Session，用于标识这个用户，并且跟踪用户，这样才知道购物车里面有什么，这个session是保存在服务器端的，有一个唯一标识。

工作原理：

（1）、浏览器发送请求时，服务器首先会读取请求头中的session信息，若果没有找到Session信息或者本地检索不到次SessionID,就新生成一个SessionID,存储到服务器硬盘或者内存中。

（2）、浏览器收到响应，会将这个返回的SessionID在本地也保存一份，供下次请求使用，Session保存在本地的其中一种实现方案是保存在Cookie上，但实际上Cookie并不是Session保存的唯一解决方案，使用URL重写的方式也可以（把SessionID拼接在URL后面），或者放在一个隐藏的表单字段里面。Servlet还提供了HttpSession接口，该接口提供了一种跨多个页面请求或访问网站识别用户以及存储有关用户信息的方式。用过HttpServletRequest的公共方法getSession()来获取HttpSession对象。

Servlet API编程常用类和接口

1、javax.servlet.Servlet接口

​ Servlet抽象集是javax.servlet.Servlet接口，它规定了必须由Servlet类实现Servlet引擎识别和管理方法集，该接口的基本目标是提供声明周期，init、service、destroy等。

2、javax.servlet.GenericServlet类

​ ServletAPI提供了Servlet接口的直接实现，成为GenericServlet。此类提供了service()方法外所有接口中方法的默认实现，所以说只要简单地扩展GenericServlet就可以编写一个基本的Servlet。除了Servlet接口外，GenericServlet也实现了ServletConfig接口，处理初始化参数和Servlet上下文，提供了授权传递到inti()方法中的ServletConfig对象的方法。

3、javax.servlet.http.HttpServlet类

​ ServletAPI允许扩展到其他协议，但最终所有的Servlet均在web环境下实施操作，只有几种Servlet直接扩展了GenericServlet，对Servlet更一般的操作是扩展其HTTP子类HttpServlet。

​ HttpServlet类通过调用指定到HTTP请求方法的方法实现service(),即对DELETE、HEAD、GET、OPTIONS、POST、PUT、REACE，分别调用doDelete()、doHEAD()、doGet()、doOptions()、doPost()、doPut()、doTrace()方法。

4、javax.servlet.ServletRequest类

ServletRequest接口封装了客户端请求的细节，它与协议无关，并有一个指定HTTP的子接口。

ServletRequest主要处理的是：

  • 找到客户端的主机和IP地址
  • 检索请求参数
  • 取得和设置属性
  • 取得输入流和输出流

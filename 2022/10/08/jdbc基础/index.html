JDBC常用类和接口

JDBC在jdk中被定义，由一组用Java语言编写的类和接口组成，大致分为两种：针对java程序员的JDBC API和针对数据库开发商底层的JDBC Driver API,可以为多种关系型数据库提供统一的访问方法。

Connection接口

​ Connection接口在java.sql包中，是与数据库连接的对象，只有获得特定的数据库连接对象，才能访问数据库进行数据库操作。

​ Connection接口要求所有实现此接口的容器,必须提供至少两种构造方法：无参构造方法，参数为Connection类的构造方法。后者需要创建一个具有和参数包含元素相同的新集合，以此来进行集合的复制。Java中所有的API均遵守此规则。

方法签名 功能描述
createStatement() 创建一个Statement对象
createStatement(int resultSetType, int resultSetConcurrency) 创建一个Statement对象，同时生成具有给指定类型、并发性和可保存性的resultSet对象。
prepareStatement(String sql) 创建预处理对象PreparedStatement对象
isReadOnly() 查看当前Connection对象的读取模式是否为只读模式
setReadOnly(boolean readOnly) 设置当前Connection对象的读取模式，默认是非只读模式
commit() 使所有上一次提交、回滚后进行的更改成为持久更改，并释放Connection对象当前持有的所有数据库锁
rollback() 取消在当前事务进行的所有更改，并释放次Connection对象当前持有的所有数据库锁
close() 立即释放次Connection对象的数据库和JDBC资源，而不是等待它们被自动释放。

DriverManager类

DriverManager类中包含与数据库交互操作的方法，该类中的方法全部有数据库厂商提供。它的方法签名及其功能描述：

方法签名 功能描述
getConnection(String url, String user, String password) 指定三个入口参数（连接数据库url,用户名,密码）来获取与数据库的连接。
setLoginTimeout(int seconds) 获取驱动程序试图登录到某一数据库时可以等待的最长时间，以s为单位
println(String message) 将一条消息输出到当前JDBC日志流中

Statement接口

Statement接口是Java程序执行数据库操作的重要接口，用于已经建立数据库连接的基础之上，向数据库发送要执行的SQL语句，主要用于执行不带参数的简单SQL语句。它的方法签名及其功能描述：

方法签名 功能描述
execute(String sql) 执行静态的select语句，该语句可能返回多个结果集。
executeQuery(String sql) 执行给定的sql语句，该语句返回单个ResultSet对象。
clearBatch() 清空此statement对象的当前SQL命令列表。
executeBatch() 将一批命令提交给数据库来执行，如果全部命令执行成功，则返回更新计数组成的数组，数组元素的排序与sql语句的添加顺序对应。
addBatch( String sql ) 将给定的SQL命令添加到此statement对象的当前命令列表中，如果驱动程序不支持批量处理，将抛出异常。
close() 释放statement实例占用的数据库和JDBC资源

PreparedStatement接口

PreparedStatement接口位于java.sql包中，它继承了Statement，但是preparedStatement相比有很大不同。

  • preparedStatement对象是经过预编译的，所以其执行速度比statement对象速度快
  • Preparedstatement继承了statement功能之外，添加了一整套方法，用于设置发送给数据库以取代参数占用符的值，可以有效避免SQL注入或攻击。

它的方法签名及其功能描述：

方法签名 功能描述
setInt(int parameterIndex, int x) 将指定位置的参数设置为int值
setLong(int parameterIndex, long x) 将指定位置的参数设置为long值
setFloat(int parameterIndex, float x) 将指定位置的参数设置为float值
setDouble(int parameterIndex, double x) 将指定位置的参数设置为double值
setDate(int parameterIndex, java.sql.Date x) 将指定位置的参数设置为date值
setString(int parameterIndex, String x) 将指定位置的参数设置为String值
executeUpdate() 执行前面包含的参数的动态Insert、update或delete语句
setNull (int parameterIndex, int sqlType, String typeName) 将指定位置的参数设置为SQL NULL
executeQuery() 在此PreparedSatement对象执行SQL查询，并返回该查询生成的ResultSet对象
clearParameters() 清除当前所有参数值

ResultSet接口

result接口是数据库结果集的结果表，通常是通过查询数据库的语句生成的，它主要提供了常用的从当前行检索列值得获取方法（getBoolean、getLong等），它的方法签名及其功能描述：

方法签名 功能描述
getInt() 以int形式获取此ResultSet对象的当前行的指定列值。如果列值是null，则返回值是0
getFloat() 以float形式获取此ResultSet对象的当前行的指定列值。如果列值是null，则返回值是0
getDate() 以date形式获取此ResultSet对象的当前行的指定列值。如果列值是null，则返回值是null
getBoolean() 以boolean形式获取此ResultSet对象的当前行的指定列值。如果列值是null，则返回值是null
getString() 以String形式获取此ResultSet对象的当前行的指定列值。如果列值是null，则返回值是null
getObject() 以Object形式获取此ResultSet对象的当前行的指定列值。如果列值是null，则返回值是null
first() 将指针移到当前记录的第一行
last() 将当前指针移到当前记录的最后一行
next() 将指针向下移一行
beforeFirst() 将指针移到集合的开头（第一行位置）
afterLast() 将指针移到集合的尾部（最后一行位置）
absolute(int index) 将指针移到ResultSet给定编号行
isFirst() 判断指针是否位于当前ResultSet集合的第一行，如果是，返回true，否则返回false
isLast() 判断指针是否位于当前ResultSet集合的最后一行，如果是，返回true，否则返回false
updateInt() 用int值更新指定列
updateFloat() 用float值更新指定列
updateLong() 用long值更新指定列
updateString() 用String值更新指定列
updateObject() 用Object值更新指定列
upDateNull() 将指定列值修改为null
updateDate() 用Date值更新指定列
updateDouble() 用double值更新指定列
getRow() 查看当前行的索引号
insertRow() 将插入行的内容插入数据库
updateRow() 将当前行的内容同步到数据表
deleteRow() 删除当前行，但并不同步到数据库中，而是在执行close()方法后同步到数据库

JDBC编程

使用JDBC连接访问数据库需要经过以下几个步骤。

  1. 加载数据库驱动
  2. 建立数据库的连接
  3. 向数据库发送SQL命令
  4. 处理数据库的返回结果集
  5. 断开与数据库连接

加载数据库驱动

​ 不同的数据库其数据库驱动类是不同的，例如，MySQL数据库的驱动类是com.mysql.cj.jdbc.Driver(老版驱动类com.mysql.jdbc.Driver),而Oracle数据库的驱动类是Oracle.jdbc.driver.OracleDriver。未避免加载失败导致意外错误，一般用==try-catch语句包围来捕获异常==。

向数据库发送SQL命令

使用SQL语句对数据库进行操作访问，是通过对Statement对象进行封装后，发送给数据库的，Statement对象不是由Statement类创建的，而是通过Connection对象提供的方法来创建各种Statement对象。

  1. CreateStatement()方法：用于创建一个基本的Statemen对象，主要用于执行静态SQL语句。
  2. PreparedSatement(String sql)方法：根据参数化的SQL语句创建一个经过预编译的PreparedStatement独享，主要用于执行动态SQL。
  3. preparedCall(String sql)方法：创建一个CallableSatement对象，主要用于执行存储过程。

获取到不同的Statement对象后，调用该对象的不同方法来执行不同的SQL语句，多有的Statement对象都有以下3种方法：

  • ResultSet executeQuery(String sql) :专用于查询
  • int executeUpdate(String sql)：执行DDL、DML语句，前者返回0，后者返回受影响的行数。
  • Boolean execute(String sql)：可以执行任何SQL语句。如果执行了查询语句，则返回true,如果执行了DDL、DML语句，则返回false。 如果true,可以通过该Statement对象的getResultSet()方法获取结果对象；如果false，可以通过该Statement对象的getUpdateCount()方法获得受影响的行数。

==需要try-catch捕获异常==

处理数据库的返回结果集

两种情况：
1、执行更新返回的是本次操作影响到的记录数。
2、执行查询返回的结果是一个ResultSet对象。

SQL查询结果都是经过ResultSet封装过的，读取ResultSet结果集数据方法主要是getXXX()。

断开与数据库的连接

所有操作都是进行完后要关闭JDBC资源，以防止系统资源的浪费，但是关闭的顺序和刚开定义对象时后相反。

  1. 关闭结果集，rs.close()。
  2. 关闭Statement对象，stmt.close()。
  3. 关闭连接，con.close()。
